# Анализ задачи

## 1. Scope (уточнение)

Задача: описать функциональность **напоминания клиенту о предстоящей записи** — автоматическое Telegram-уведомление, отправляемое за 2 часа до начала записи.

## 2. Проверка правил

**Затронутые ограничения и правила:**

| ID | Релевантность |
|----|---------------|
| `C-003` | Telegram Bot API лимиты — при массовой отправке напоминаний нужна очередь |
| `C-002` | Персональные данные клиента (телефон, имя) не должны попадать в логи |
| `C-001` | Напоминание отправляется только по подтверждённым записям |
| `C-006` | Если бронирование было перенесено — напоминание должно учитывать новое время |

**Из глоссария — прямое указание:**

> `#12 Напоминание (Reminder)` — ⚠️ В текущей версии **НЕ реализовано**. Планируется: push за N часов до start_at. Техдолг.

Задача описывает **новую функциональность**, а не документирование существующей — это BA-зона. ✅

---

## 3. Уточняющие вопросы перед артефактом

Прежде чем создать user story, мне нужны ответы на несколько вопросов:

**Обязательные (блокируют написание AC):**

1. **Канал доставки:** только Telegram? Или также SMS / push на веб? Глоссарий упоминает только Telegram-бот — подтверждаем только его?
2. **Триггер подписки:** клиент получает напоминание только если у него есть активная Telegram-подписка (`Subscription` типа `CLIENT_BOOKING`)? Что делать, если подписки нет — молча пропустить или логировать?
3. **Статусы-триггеры:** напоминание отправляется только по записям в статусе `confirmed`? Или также по `pending`?
4. **Точность времени:** "за 2 часа" — это жёстко 2 часа или настраиваемый параметр (N часов) на уровне бизнеса / платформы?
5. **Повторная отправка:** если клиент перенёс запись (reschedule подтверждён) — напоминание пересчитывается по новому времени? Старое отменяется?
6. **Записи в ближайшие 2 часа от момента создания:** если клиент записался за 1 час до сеанса — напоминание не отправляется (уже прошло окно) или отправляется немедленно?

**Желательные (влияют на edge cases):**

7. Нужно ли уведомление провайдеру (мастеру/бизнесу) о предстоящих записях — или только клиенту?
8. Если запись отменена (`cancelled`) после того, как напоминание уже поставлено в очередь — отправка отменяется?
