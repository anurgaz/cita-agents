## Метаданные

## Формулировка

**Как** владелец бизнеса (Администратор),
**я хочу** видеть статистику бронирований за неделю и месяц
в разделе «Статистика» Telegram Mini App,
**чтобы** понимать эффективность работы бизнеса, загруженность
мастеров и причины потерь.

---

## Контекст

Провайдер управляет бизнесом через Telegram Mini App.
Сейчас он видит только список отдельных бронирований без
агрегатов. Для принятия решений (например, нанять мастера,
скорректировать расписание) нужна агрегированная картина.

Статистика строится по бронированиям, принадлежащим бизнесу
провайдера (business_id). Доступ только для владельцев с
ролью ADMIN.

Выручка рассчитывается по price_snapshot — цене,
зафиксированной в момент создания бронирования (C-010, BR-005).

## Acceptance Criteria

### AC-1: Переключение периода (неделя / месяц)

```
Given: провайдер открыл раздел «Статистика» в Telegram Mini App
When: страница загрузилась
Then: отображается переключатель периода с двумя значениями —
      «Неделя» и «Месяц»
  And: по умолчанию выбран период «Неделя»
  And: «Неделя» = текущая календарная неделя (Пн 00:00 — Вс 23:59)
  And: «Месяц» = текущий календарный месяц (1-е число 00:00 —
       последний день 23:59)
```

### AC-2: Отображение ключевых метрик

```
Given: провайдер выбрал период
When: данные загружены
Then: отображаются четыре карточки метрик:
      1. «Всего записей» — count(bookings) за период,
         все статусы кроме deleted
      2. «Выполнено» — count(status = 'completed')
      3. «Отменено» — count(status = 'cancelled')
      4. «Ожидают» — count(status = 'pending')
  And: каждая карточка показывает абсолютное число
  And: карточка «Отменено» дополнительно показывает процент
       от «Всего записей» (cancellation rate)
```

### AC-3: Загрузка мастеров

```
Given: провайдер выбрал период
When: данные загружены
Then: отображается список мастеров бизнеса
  And: для каждого мастера показывается:
       — имя мастера
       — количество бронирований за период
         (статусы: confirmed + completed)
  And: список отсортирован по убыванию количества
       бронирований
  And: если у мастера 0 бронирований за период —
       он всё равно присутствует в списке со значением «0»
```

### AC-4: Отображение выручки

```
Given: провайдер выбрал период
When: данные загружены
Then: отображается карточка «Выручка»
  And: значение = SUM(price_snapshot) по бронированиям
       со статусом 'completed' за период
  And: формат: «45 000 ₸»
  And: если completed-записей нет — отображается «0 ₸»
```

### AC-5: Авторизация и изоляция данных

```
Given: провайдер открывает раздел «Статистика»
When: запрос на получение статистики выполняется
Then: возвращаются данные ТОЛЬКО по business_id,
      связанному с JWT-токеном текущего владельца
  And: если токен истёк (C-008) — отображается экран
       «Сессия истекла, войдите снова»
  And: владелец с ролью, отличной от ADMIN,
       не имеет доступа к разделу (403)
```

### AC-6: Состояния загрузки и ошибок

```
Given: провайдер открыл раздел «Статистика»
When: данные ещё загружаются
Then: отображается skeleton-loader вместо карточек

When: запрос завершился ошибкой (5xx или таймаут)
Then: отображается сообщение «Не удалось загрузить статистику»
  And: присутствует кнопка «Повторить»
  And: при нажатии «Повторить» — запрос выполняется заново

When: за выбранный период нет ни одного бронирования
Then: карточки отображаются со значением «0»
  And: список мастеров отображается (все с «0»)
  And: НЕ отображается пустой экран без данных
```

## Сценарии

### Happy Path

1. Провайдер открывает Telegram Mini App → переходит в раздел
   «Статистика»
2. По умолчанию загружается статистика за текущую неделю
3. Провайдер видит: 23 записи всего, 15 выполнено, 5 отменено
   (21.7%), 3 ожидают
4. Видит выручку: 87 500 ₸ (по completed)
5. Видит список: Анна — 10, Дмитрий — 5 (confirmed + completed)
6. Нажимает «Месяц» — данные обновляются без перезагрузки страницы

### Альтернативный сценарий: новый бизнес

1. Провайдер зарегистрировался на этой неделе
2. Открывает «Статистику» — период «Неделя»
3. Все метрики = 0, выручка = 0 ₸
4. Список мастеров пуст (если мастера ещё не добавлены)
5. Отображается подсказка: «Записей пока нет»

### Edge Case: перекос нагрузки из-за автоназначения (C-011)

1. В бизнесе 3 мастера, но алгоритм автоназначения (C-011)
   направляет все записи к первому мастеру
2. В статистике загрузки: Мастер А — 20 записей, Мастер Б — 0,
   Мастер В — 0
3. Система отображает данные корректно — без сглаживания
4. Провайдер видит проблему распределения нагрузки

### Edge Case: истёкший токен

1. Провайдер открывает «Статистику» через 8+ дней после логина
2. Запрос возвращает 401
3. Отображается: «Сессия истекла, войдите снова» (C-008)

## Out of Scope

- Сравнение с предыдущим периодом («+15% к прошлой неделе»)
- Произвольный диапазон дат (date picker)
- Детализация по конкретным услугам
- Статистика no-show (не реализована в системе — отсутствует
  статус `no_show`, см. glossary #20)
- Рейтинги и отзывы (не реализованы, glossary #21)
- Напоминания об аналитике (push-уведомления с итогами недели)
- Экспорт данных (PDF, Excel, CSV)
- Статистика для роли SuperAdmin (все бизнесы) — отдельная
  user story
- Сравнение мастеров по выручке на мастера
- История периодов (прошлая неделя, прошлый месяц)
