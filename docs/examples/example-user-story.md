# US-003: Запись к мастеру через Telegram Mini App

## Meta
- **Приоритет:** Must
- **Эпик:** Онлайн-запись через Telegram
- **Агент-автор:** BA
- **Дата:** 2026-03-20

## User Story
**Как** клиент,
**я хочу** записаться к мастеру через Telegram Mini App,
**чтобы** быстро забронировать удобное время без звонков и регистрации.

## Контекст
Telegram — основной канал коммуникации для целевой аудитории (Казахстан). Mini App позволяет записаться
в 3-4 нажатия без установки отдельного приложения. Клиент переходит по ссылке от бизнеса
(QR-код или deep link в чате) и попадает сразу на страницу записи.

## Acceptance Criteria

### AC-1: Авторизация через Telegram
- **Given** клиент открыл Mini App через deep link `https://t.me/{bot}?startapp={slug}`
- **When** Mini App загрузилась
- **Then** клиент автоматически авторизован (initData -> JWT), видит название бизнеса и список услуг

### AC-2: Выбор услуги и мастера
- **Given** клиент авторизован и видит список услуг
- **When** клиент выбирает услугу
- **Then** отображается список мастеров, которые выполняют эту услугу, с возможностью выбрать "Любой мастер"

### AC-3: Выбор даты и времени
- **Given** клиент выбрал услугу (и опционально мастера)
- **When** клиент выбирает дату
- **Then** отображаются только свободные слоты с учетом расписания мастера, перерывов и существующих записей

### AC-4: Подтверждение записи
- **Given** клиент выбрал услугу, дату и время
- **When** клиент вводит имя и телефон и нажимает "Записаться"
- **Then** создается запись в статусе `pending`, клиент видит подтверждение, провайдер получает Telegram-уведомление

### AC-5: Автоназначение мастера
- **Given** клиент выбрал "Любой мастер"
- **When** создается запись
- **Then** система автоматически назначает мастера, у которого данный слот свободен и который выполняет выбранную услугу

### AC-6: Фиксация цены
- **Given** услуга стоит 5000 тенге на момент записи
- **When** создается запись
- **Then** в записи сохраняется `price_snapshot = 5000`, даже если цена потом изменится

## Business Rules
- **BR-001:** Автоназначение мастера — выбирается первый свободный мастер с нужной услугой
- **BR-002:** Начальный статус записи — `pending` (требует подтверждения провайдером)
- **BR-003:** Слот проверяется на доступность перед созданием записи
- **BR-005:** Цена фиксируется как `price_snapshot` из `service.price`
- **BR-008:** Client upsert — если телефон уже есть в базе бизнеса, обновляется имя
- **SR-001:** Расписание мастера приоритетнее расписания бизнеса
- **SR-005:** Перерывы блокируют слоты
- **NR-001:** Провайдер получает Telegram-уведомление о новой записи с inline-кнопками

## Data Requirements
| Поле | Тип | Обязательное | Источник |
|------|-----|:---:|----------|
| service_id | Integer | да | Выбор клиента (список из GET /public/{slug}/services) |
| master_id | Integer | нет | Выбор клиента или auto-assign |
| date | String (YYYY-MM-DD) | да | Выбор клиента (календарь) |
| time | String (HH:MM) | да | Выбор клиента (из GET /public/{slug}/slots) |
| first_name | String | да | Ввод клиента |
| phone | String | да | Ввод клиента (формат: +7XXXXXXXXXX) |
| notes | String | нет | Ввод клиента (комментарий) |

## UI/UX Notes
- Haptic feedback при нажатии кнопок (impactOccurred medium)
- MainButton Telegram используется для "Записаться" на финальном шаге
- BackButton для навигации назад по шагам
- После успешной записи — notificationOccurred success + экран подтверждения

## Out of Scope
- Оплата через Telegram Payments
- Запись на несколько услуг одновременно
- Выбор конкретного рабочего места/кабинета
- Повторная запись (запись на основе предыдущей)
- Push-уведомления в Mini App о статусе записи (уведомления идут через бота)
